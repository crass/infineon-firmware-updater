◆ deviceTypeChange

unsigned int BITFIELD_NEW_TPM_FIRMWARE_INFO::deviceTypeChange

Firmware update will change the TPM device type. For example:

  • Firmware update will update a TPM1.2 to a TPM2.0.
  • Firmware update will update a TPM2.0 to a TPM1.2.

◆ factoryDefaults

unsigned int BITFIELD_NEW_TPM_FIRMWARE_INFO::factoryDefaults

Firmware update will update the TPM and reset the TPM to factory defaults. All data and state from the old TPM firmware will be lost. The appearance of this depends on the actual source and target firmware of the firmware update and can occur in both, cross version and normal update paths.
